## 📖 Grundkonzept: Der Ablauf hinter den Kulissen

Wenn du eine Webseite besuchst, passiert viel mehr als du siehst.
Hier ist der komplette Ablauf:

```
1. Du tippst eine URL in den Browser
        ↓
2. Browser sendet einen REQUEST (Anfrage)
        ↓
3. Python-Server empfängt die Anfrage
        ↓
4. Server verarbeitet die Anfrage
   → Welche Route wurde aufgerufen?
   → Werden Daten aus einer Datenbank gebraucht?
        ↓
5. Server erstellt eine HTML-Seite (Template)
        ↓
6. Server sendet eine RESPONSE (Antwort)
        ↓
7. Dein Browser zeigt die fertige Seite an ✅
```

---

## 🔑 Die 5 wichtigsten Begriffe

| Begriff | Einfache Erklärung | Beispiel |
|---|---|---|
| **Framework** | Werkzeugkasten für Entwickler | Flask, Django |
| **Route** | Adresse einer Unterseite | `/home`, `/login` |
| **Request** | Anfrage vom Browser | *"Zeig mir die Startseite!"* |
| **Response** | Antwort vom Server | *"Hier ist deine HTML-Seite!"* |
| **Template** | HTML-Vorlage mit Platzhaltern | `Hallo, {{ name }}!` |

---

## 🛠️ Die Hauptteile einer Python-Webseite

### 1️⃣ Das Framework
> Das Framework ist dein Hilfswerkzeug – es übernimmt die schwere Arbeit.

Zwei beliebte Frameworks:
- 🟢 **Flask** → klein, einfach, perfekt für Anfänger
- 🔵 **Django** → groß, mächtig, für komplexe Projekte

---

### 2️⃣ Die Route
> Eine Route verbindet eine URL mit einer Python-Funktion.

```python
# Beispiel: Flask-Route
@app.route("/home")
def startseite():
    return "Willkommen auf meiner Seite!"
```

➡️ Jemand besucht `/home` → Python führt `startseite()` aus

---

### 3️⃣ Das Template
> Templates sind HTML-Vorlagen, in die Python Daten einfügt.

```html
<!-- Beispiel: Template mit Platzhalter -->
<h1>Hallo, {{ benutzername }}!</h1>
<p>Du hast {{ nachrichten }} neue Nachrichten.</p>
```

➡️ Python füllt `{{ benutzername }}` mit echten Daten

---

## 💡 Merkhilfe: Das Restaurant-Prinzip

Stell dir eine Webseite wie ein Restaurant vor:

| Restaurant | Webseite |
|---|---|
| 🧑 Gast bestellt | Browser sendet **Request** |
| 📋 Speisekarte | **Routes** (was ist verfügbar?) |
| 👨‍🍳 Küche kocht | **Server** verarbeitet Anfrage |
| 🍽️ Essen wird serviert | Server sendet **Response** |
| 🏠 Esszimmer | **Template** (wie es aussieht) |

---

## ✅ Zusammenfassung

```
Browser → Request → Server (Route) → Template → Response → Browser
```

**Die 3 wichtigsten Punkte:**
1. 🔄 Webseiten funktionieren nach dem **Request-Response-Prinzip**
2. 🗺️ **Routen** bestimmen, welche Funktion aufgerufen wird
3. 🎨 **Templates** trennen Design und Programmlogik
